# Gantt Chart Generator

> ブラウザだけで動くガントチャート生成ツール。タスク・工数・担当者を設定すると、リソース制約付きで自動スケジューリングし、ガントチャートを描画します。

**[→ gantt-generator.html をダウンロードして開くだけで使えます](gantt-generator.html)**

---

## 特徴

- **ゼロインストール** — 単一 HTML ファイル。ブラウザで開くだけ
- **自動スケジューリング** — LPT（最長パイプライン優先）アルゴリズムで担当者を自動配置
- **リソース制約対応** — 担当者ごとの担当工程・参画開始日・稼働率を考慮
- **3段階ズーム** — 日 / 週 / 月 単位でガントチャート表示を切り替え
- **休日対応** — 国民の祝日（API自動取得 or バンドル済みデータ）＋ 会社指定休業日
- **エクスポート** — HTML スナップショット / PNG / JSON（設定の保存・読み込み）
- **設定は自動保存** — localStorage に保存されるため、ページを閉じても設定が残る

---

## 使い方

### 1. ファイルを開く

`gantt-generator.html` をブラウザで開きます（ダブルクリックで OK）。

### 2. 設定を入力する

左サイドバーの4つのタブで設定します。

| タブ | 設定内容 |
|------|---------|
| **工程タイプ** | フェーズの種類（要件定義、設計開発など）と担当チーム・色 |
| **アイテム** | 開発項目ごとのフェーズ・工数・担当者割付 |
| **担当者** | 担当者の担当可能工程・参画開始日・稼働率 |
| **プロジェクト設定** | 開始日・リリース日・評価期間・表示単位・休日 |

### 3. スケジュールを確認する

設定を変更するたびに右ペインのガントチャートとサマリー表がリアルタイムで更新されます。

### 4. エクスポートする

| ボタン | 出力内容 |
|--------|---------|
| **HTML 出力** | 現在のガントチャートを静的 HTML として保存（共有・配布向け） |
| **PNG 出力** | ガントチャートカードを PNG 画像として保存 |
| **JSON 保存** | 設定を JSON ファイルとして保存（バックアップ・チーム共有） |
| **JSON 読込** | 保存した JSON を読み込んで設定を復元 |

## スケジューリングの仕組み

### アルゴリズム

LPT（Longest Processing Time）方式を採用しています。残りパイプライン工数が多いタスクを優先的に担当者に割り当てることで、全体の終了時刻を短縮します。

```
1. 各タスクに priority = 残パイプライン工数（自タスク以降の合計）を付与
2. 1営業日ずつシミュレーション:
   a. 先行フェーズ完了 → 後続フェーズを ready キューへ
   b. ready キューを priority 降順でソート
   c. 担当者の空き・制約を確認して割り当て
3. 全タスク完了 or リリース日+猶予 まで繰り返し
```

### フェーズの設定オプション

| オプション | 説明 |
|------------|------|
| `allowedPeople` | このフェーズを担当できる人を絞り込む（省略時は自動選択） |
| `requireAll` | `allowedPeople` に指定した全員が同時に着手する |
| `background` | 工数を消費しない並行作業（ハードウェア調達・外注待ちなど） |
| `fixedStart` | 特定日に強制開始する |

## 設定ファイル（JSON）の形式

```jsonc
{
  "projectName": "v2026.07 リリーススケジュール",
  "startDate": "2026-04-20",
  "releaseDate": "2026-07-15",
  "evalPeriod": { "value": 4, "unit": "weeks" },  // "days" | "weeks" | "months"
  "ganttUnit": "weeks",                            // "days" | "weeks" | "months"
  "holidays": {
    "national": ["2026-04-29", "2026-05-03"],      // 国民の祝日
    "company":  ["2026-12-29", "2026-12-30"]       // 会社指定休業日
  },
  "phaseTypes": [
    { "name": "要件定義", "team": "D&T", "color": "#3B82F6" }
  ],
  "people": [
    {
      "name": "Tanaka",
      "team": "Dev",
      "phases": ["設計開発"],
      "availableFrom": null,   // null = プロジェクト開始から参画
      "utilization": 1.0,      // 稼働率（0.5 = 50%）
      "note": ""
    }
  ],
  "items": [
    {
      "name": "機能A開発",
      "category": "コア機能",
      "note": "",
      "phases": [
        { "type": "要件定義", "days": 5 },
        { "type": "設計開発", "days": 10, "allowedPeople": ["Tanaka"] }
      ]
    }
  ],
  "evalPhase": { "name": "リリース評価", "color": "#8B5CF6" }
}
```

サンプル設定ファイルは [`examples/v2026.07-config.json`](examples/v2026.07-config.json) を参照してください。

## 休日データについて

- **バンドル済みデータ**: 2024〜2028年の日本の祝日を内蔵しています
- **自動取得**: 「プロジェクト設定」タブの「自動取得」ボタンで [holidays-jp API](https://holidays-jp.github.io) から最新データを取得できます（ネット接続が必要）
- **会社指定休業日**: 年末年始・創立記念日などを手動で追加できます
